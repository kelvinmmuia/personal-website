================================================================================
STEP 1: CREATE GITHUB REPOSITORY
================================================================================

1. Open your browser and go to: https://github.com/new

2. Fill in these details:
   - Repository name: personal-website
   - Description: My personal portfolio - Data Scientist & Web Developer
   - Visibility: PUBLIC (important!)
   - Do NOT initialize with README

3. Click "Create repository"

4. You'll see a page with commands. Copy the HTTPS URL.
   It should look like: https://github.com/kelvinmwakamuia-cell/personal-website.git

================================================================================
STEP 2: OPEN WINDOWS COMMAND PROMPT
================================================================================

1. Press: Windows Key + R
2. Type: cmd
3. Press: Enter

You should see a black window with "C:\Users\USER>"

================================================================================
STEP 3: NAVIGATE TO YOUR PROJECT
================================================================================

In the Command Prompt, type:

    cd c:\Users\USER\CascadeProjects\personal-website

Press Enter.

You should see: C:\Users\USER\CascadeProjects\personal-website>

================================================================================
STEP 4: INITIALIZE GIT
================================================================================

Copy and paste each command one at a time. Press Enter after each.

    cmd /c git init

    cmd /c git add .

    cmd /c git commit -m "Initial commit: Personal website portfolio"

    cmd /c git branch -M main

================================================================================
STEP 5: CONNECT TO GITHUB
================================================================================

Replace the URL with your repository URL from Step 1:

    cmd /c git remote add origin https://github.com/kelvinmwakamuia-cell/personal-website.git

Press Enter.

================================================================================
STEP 6: PUSH TO GITHUB
================================================================================

    cmd /c git push -u origin main

Press Enter.

You may be asked to authenticate. Follow GitHub's instructions.
You can use:
- Personal Access Token (recommended)
- GitHub CLI
- SSH key

================================================================================
STEP 7: DEPLOY TO GITHUB PAGES
================================================================================

    cmd /c npm run deploy

Press Enter.

This will:
1. Build your website
2. Create a 'dist' folder
3. Deploy to GitHub Pages
4. Create a 'gh-pages' branch

Wait for it to complete. You should see "Published" message.

================================================================================
STEP 8: CONFIGURE GITHUB PAGES SETTINGS
================================================================================

1. Go to: https://github.com/kelvinmwakamuia-cell/personal-website/settings/pages

2. Under "Build and deployment" section:
   - Source: Select "Deploy from a branch"
   - Branch: Select "gh-pages" from dropdown
   - Folder: Select "/root"

3. Click "Save"

4. Wait 1-2 minutes for GitHub to process

================================================================================
STEP 9: VERIFY YOUR WEBSITE IS LIVE
================================================================================

Open your browser and go to:

    https://kelvinmwakamuia-cell.github.io/personal-website

================================================================================
TROUBLESHOOTING
================================================================================

PROBLEM: Website not showing at GitHub Pages URL

SOLUTION:
1. Wait 2-3 minutes after deployment
2. Hard refresh: Ctrl + Shift + R
3. Check GitHub Pages settings
4. Verify gh-pages branch exists in your repository

---

PROBLEM: Styles not loading or images broken

SOLUTION:
1. Rebuild and redeploy:
   cmd /c npm run build
   cmd /c npm run deploy

2. Hard refresh browser: Ctrl + Shift + R

---

PROBLEM: Port 5173 already in use

SOLUTION:
   cmd /c npm run dev -- --port 3000

---

PROBLEM: Git authentication fails

SOLUTION:
1. Use Personal Access Token instead of password
2. Go to: https://github.com/settings/tokens
3. Generate new token with 'repo' scope
4. Use token as password when prompted
